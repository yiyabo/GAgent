# PhageAgent 开发流程：从 Prompt 到 Harness

## Prompt：定义 Agent 的行为边界

我们最早的工作从 prompt 开始。作为一个面向噬菌体生物信息学的 AI agent，PhageAgent 需要理解领域术语、选择合适的分析工具、并按照规范的流程执行任务。系统提示词承担的就是"性格塑造"的角色——它告诉模型自己是谁、能调用哪些工具、在面对不同类型的请求时应该如何思考和行动。

但很快我们发现，仅靠 prompt 的静态指令远远不够。当任务变得复杂——比如同时涉及生命周期分类和宿主范围预测——不同指令之间开始冲突，模型要么顾此失彼，要么答非所问。更关键的是，prompt 的上下文窗口是有限的，我们没办法把整个噬菌体知识体系、873,718 条基因组的注释信息、以及十几种分析工具的使用规范全部塞进一段提示词里。模型只能依赖预训练知识去回答专业问题，而这些知识往往是模糊的、过时的、甚至是编造的。

这让我们意识到，prompt 能定义 agent 的行为边界，但无法提供它在每一次推理中真正需要的"当下认知"。于是我们开始思考：怎样在模型推理之前，动态地把最相关的信息送到它面前？

## Context：RAG + A-mem 构建动态认知

### RAG：外部知识检索

这个问题的答案是构建一个外部知识层——让模型在每次推理前，检索到与当前任务最相关的领域知识，而不是凭记忆作答。

我们构建了一个面向噬菌体生物信息学的文献知识图谱，收录了超过 10 万篇相关论文，涵盖噬菌体基因组学、宿主互作、噬菌体治疗、微生物组、病毒分类学等方向。关键在于结构化：每篇论文按语义拆解为摘要层（核心发现和结论）、方法层（工具、参数、数据处理流程）和任务层（实际解决的生物信息学问题类型）。实体之间通过关系边相连——噬菌体与宿主之间有感染关系，工具与任务之间有适用关系，数据集与实验之间有来源关系——形成一张可查询的语义网络。

当研究者提出问题时，agent 不再凭训练语料里模糊的印象去回答，而是先在图谱中检索相关节点，把方法描述、参数规范和领域上下文注入当前的 context window，再让模型基于实证信息进行推理。这就是 RAG 的核心作用：把一个"什么都知道一点但什么都不确定"的通用模型，变成一个"当下就能查到确切答案"的领域专家。

### A-mem：Agent 自身的操作记忆

但 RAG 解决的是"查得到"的问题，还有一类知识是查不到的——它来自 agent 自身的使用经验。

比如：PhageScope 的 API 提交分析任务时，`modulelist` 必须是 JSON 对象格式而不是数组格式，`phageid` 和 `phageids` 必须同时提供。这些细节不会写在任何论文里，它们是 agent 在实际调用工具的过程中，通过失败、修正、再成功的循环积累下来的操作经验。

为此我们引入了 A-mem（Agentic Memory），一套面向 agent 自身的长期记忆机制。它记录的是行为轨迹：哪些工具调用成功了、用了什么参数、失败的原因是什么。更关键的是，这套记忆会经历"自我净化"——早期不成熟的调用模式随着更多成功经验的积累被逐渐淘汰，反复验证有效的工具链路和参数组合则被强化保留。这有点像一个新手研究员在实验室里摸索后形成的肌肉记忆：不需要每次翻手册，因为已经"知道"正确的操作手感了。

A-mem 与我们系统的契合度很高。PhageAgent 整合了 35 种以上的生物信息学工具，每种都有自己的参数规范和调用约束，中间任何一个环节的参数传错都会导致整条链路失败。有了 A-mem，agent 能直接调取过往成功的调用模式，减少不必要的试错循环——一个"越用越准"的工具调用直觉。

至此，context 层形成了两条互补的信息通路：RAG 从外部文献检索领域知识，回答"这个问题在学术上怎么解"；A-mem 从自身经验检索操作知识，回答"这个任务在实践中怎么做"。

## Harness：从"能想到"到"做得到"

到这一步，agent 已经具备了充分的认知基础。但一个关键问题浮出水面——它依然经常"说对了却做错了"。

这要从 LLM 的本质说起。大语言模型归根结底是一个条件概率模型：给定上下文，每一个 token 都是对"下一个最可能的词"的采样。Prompt 在收窄概率空间，Context 在注入先验，但无论概率空间被压缩得多好，它终究是一个概率分布，而不是一个确定性程序。模型可能在 95% 的情况下选对了工具，却在第三步突然跳到不相关的操作；可能准确规划了十步流程，却在执行到一半时"忘记"前五步重新开始；单次推理给出完美答案，但无法保证下一次还能复现。

Prompt 和 context 解决的是"想"的问题。但一个真正可用的 agent 必须能"做"——可靠地、可追溯地、可恢复地做完一整件事。Harness 不改变模型的认知能力，而是在概率性输出之外叠加一层确定性的执行骨架，确保"想到"变成"做到"。

### ReAct 循环：把发散的思维变成收敛的行动

模型每一轮推理被约束在 Think → Act → Observe 的三拍节奏中：先生成思维链，明确当前该做什么；然后执行一个具体动作——调用工具或发起查询；最后观察结果，把真实数据整合进上下文再进入下一轮。循环最多 24 轮，第 8 轮后引导收敛，第 15 轮后要求最终答案。每一次 Observe 都是用真实世界的反馈把模型拉回正轨，防止长链推理中逐渐偏离目标。

### Plan DAG 调度：把模糊意图变成可执行的依赖图

当研究者说"帮我复现这篇 Nature 论文的核心发现"，这对模型是一个巨大的概率空间。Plan DAG 通过递归广度优先分解，把模糊意图拆成任务树，再通过语义相似度合并冗余节点、验证无环性，编译成有向无环图。每个节点是最小可执行单元，独立节点并行，有依赖的按拓扑序串行。研究者随时可以插手编辑，系统增量重规划受影响的子图，同时保留已完成分支。

### Checkpoint-Resume：把脆弱的长链变成可断点续跑的流程

复杂流水线可能跑好几天——Nature 复现中一个 metaFlye 组装就四天。我们在 DAG 节点粒度实现了状态持久化：每个节点完成后输出写入数据库，恢复时跳过已完成节点，失败节点进入自动修复循环。这让概率模型驱动的 agent 获得了确定性系统才有的容错能力。

### 沙箱隔离：把不可预测的输出限制在安全边界内

模型生成的代码在受约束的 Docker 容器中运行——只读数据挂载、512MB 内存上限、禁用网络。计算密集型任务通过 SSH 发送到远程服务器，agent 自动管理文件传输和状态轮询。概率模型永远可能产生意外输出，沙箱确保这些意外不会造成不可逆后果。

### 错误自动修复：把失败从终点变成中间状态

工具调用失败时，系统自动触发深度思考重试：把错误信息、工具日志和一套预置的生物信息学修复工具包（SeqKit、BLAST、Prodigal、HMMER、CheckV、geNomad）注入上下文，让模型自己诊断并生成修复方案。不是消除错误的可能性——对概率模型这不可能——而是把每一次错误变成自我修正的机会。

---

这些机制合在一起，构成了 PhageAgent 的 harness 层。它的本质是一个执行逻辑：LLM 不是一个可靠的执行引擎，它是一个强大但不确定的推理引擎。Prompt 和 context 让它尽可能"想对"，harness 则保证——即使偶尔想错了——整个系统依然能沿正确方向前进，最终交付可验证、可追溯、可复现的结果。
