# 多源关联网络的可信度评估与生态信号挖掘

## 基于PhageScope 873K噬菌体序列数据集的噬菌体-宿主互作研究方案（重构版）

---

## 一、研究背景与核心科学问题

### 1.1 噬菌体-宿主互作预测的现状

当前噬菌体-宿主互作（Phage-Host Interaction, PHI）数据库主要依赖计算方法进行大规模预测，包括基于k-mer频率的方法（如VirHostMatcher）、基于CRISPR spacer匹配的方法（如CRISPRCasFinder）、基于序列同源性的方法（如BLAST-based approaches）以及基于宏基因组共现模式的方法（如WIsH）。这些方法各有优劣，但其假阳性率在实际应用中往往难以精确评估。

### 1.2 核心问题：多源异构数据的可信度困境

PhageScope作为当前最大的噬菌体元数据整合平台，汇集了来自8个公共数据库的873,718条噬菌体记录。然而，这种大规模整合带来了一个根本性问题：**不同数据源采用不同的预测方法、质量阈值和分类标准，简单合并会产生一个充满噪声的互作网络**。具体而言：

- **GOV2**（195,699条，海洋来源）和**MGV**（189,680条，肠道来源）主要基于宏基因组组装和k-mer方法预测宿主
- **IMG_VR**（177,361条）采用多种方法组合，覆盖多种生态环境
- **GPD**（142,809条）专注于肠道噬菌体，使用CRISPR spacer和共现分析方法
- 其余4个数据源（PhagesDB, GVD, EMBL, DDBJ）各有独特的方法学特征

这种异质性意味着，同一条噬菌体-宿主关联在不同数据源中可能得到矛盾的预测结果，而不同来源之间的系统性偏差可能被误读为生物学信号。

### 1.3 核心科学问题

本研究围绕以下三个关键科学问题展开：

1. **可信度问题**：跨8个数据源的计算预测噬菌体-宿主关联的可靠性如何？不同数据源的假阳性率和假阴性率是否存在系统性差异？
2. **信号分离问题**：在整合后的互作网络中，如何区分真正的生态信号（生物群落特异性的噬菌体-宿主互作模式）与方法学假象（由预测算法偏差导致的虚假模式）？
3. **生态格局问题**：噬菌体-宿主互作网络的拓扑结构在不同生态位（肠道 vs 海洋 vs 土壤）之间如何变化？这些差异是否反映了真实的生态适应性？

### 2.3 质量控制与过滤策略

采用分级过滤策略，构建分析就绪数据集：

**第一级过滤：质量阈值**
- 移除Low-quality记录（267,050条），保留High-quality + Medium-quality + Complete（共606,668条）
- 理由：Low-quality基因组可能包含严重的组装错误，导致宿主预测不可靠

**第二级过滤：元数据完整性**
- 移除缺失Host注释的记录（预计在第一级过滤后约剩余~12,000条）
- 移除缺失Taxonomy注释的记录
- 预计最终保留约567,000条记录

**第三级过滤：统计稳健性**
- 聚焦于噬菌体数 ≥ 100的宿主属，确保后续统计分析的稳健性
- 预计覆盖Top 50-80个宿主属，占总互作关系的80%以上

## 三、多源关联网络的构建与分层

### 3.1 二分网络构建

构建噬菌体-宿主属级二分网络（bipartite phage-host genus network）：

- **节点**：噬菌体序列节点（~567K）和宿主属节点（~616）
- **边**：噬菌体与宿主属之间的预测互作关系
- **边权重**：证据多重性（evidence multiplicity），即有多少个独立数据源支持该互作关系

### 3.2 多证据置信度评分（MECS）

定义多证据置信度评分（Multi-Evidence Confidence Score, MECS），综合以下三个维度：

$$MECS = w_1 \cdot S_{source} + w_2 \cdot S_{lifestyle} + w_3 \cdot S_{quality}$$

其中：
- **$S_{source}$**（来源一致性分数）：支持该互作的独立数据源数量 / 总数据源数量（0-1范围）
- **$S_{lifestyle}$**（生活方式一致性分数）：不同来源对该噬菌体生活方式预测的一致性（全部一致=1.0，部分一致=0.5，矛盾=0.0）
- **$S_{quality}$**（质量一致性分数）：不同来源对该噬菌体基因组完整性评估的一致程度
- **$w_1, w_2, w_3$**：权重参数，初始设定为 $w_1=0.5, w_2=0.2, w_3=0.3$，后续通过实验验证数据优化

### 3.3 网络层分解

将整合网络分解为多个子网络，用于后续的生态信号挖掘：

| 网络层 | 分解维度 | 子网络数 | 用途 |
|---|---|---|---|
| 来源层 | 按数据源 | 8个 | 来源可靠性比较 |
| 生态位层 | 按推断生态位 | 3-5个（肠道/海洋/多环境/土壤/其他） | 生态位特异性分析 |
| 生活方式层 | 按lifestyle | 2个（virulent/temperate） | 生活方式-生态关联 |
| 置信度层 | 按MECS阈值 | 3个（高/中/低置信度） | 网络鲁棒性分析 |

## 四、网络可信度评估框架

### 4.1 CRISPR Spacer匹配验证

CRISPR spacer序列是细菌免疫系统的分子化石，记录了历史上的噬菌体感染事件。利用这一独立证据层进行网络验证：

**数据获取**：
- 从CRISPRCasFinder、CRISPRdb和NCBI RefSeq中提取已知CRISPR阵列和个体spacer序列
- 目标：覆盖PhageScope中Top 50宿主属的参考基因组

**匹配策略**：
- 使用BLASTn将spacer序列与567K+噬菌体序列进行比对
- 严格匹配标准：identity ≥ 95%，匹配长度 ≥ 28bp，E-value < 1e-5
- PAM序列验证：检查匹配位点下游是否存在原间隔序列邻近基序

**评估指标**：
- 对每条预测的噬菌体-宿主互作边，检查是否存在CRISPR spacer证据
- 计算CRISPR验证率（CRISPR-validated rate）：有spacer证据支持的边数 / 总边数
- 按数据源分层计算CRISPR验证率，识别高可靠性数据源

### 4.2 实验验证数据集交叉验证

构建实验验证的金标准数据集，用于评估各数据源的预测性能：

**金标准来源**：
- **PhagesDB**（phagesdb.org）：培养验证的噬菌体-宿主对，约3,000+对
- **ATCC/DSMZ菌种保藏中心**：商业化的噬菌体-宿主配对
- **已发表的分离培养研究**：系统性文献检索，目标补充至≥5,000对

**分类学统一化**：
- 将实验验证对的宿主分类学映射到PhageScope使用的统一宿主属级分类
- 排除分类学不一致或模糊的配对

**性能基准测试**：
- 对8个数据源分别计算Precision、Recall和F1分数
- 计算整合网络（所有来源合并）和过滤网络（仅高置信度边）的性能
- 按宿主属分层计算性能指标，识别表现优异/不佳的分类群

### 4.3 来源可靠性排名

基于实验验证数据集和CRISPR验证结果，对8个数据源进行可靠性排名：

| 排名维度 | 指标 | 计算方式 |
|---|---|---|
| 精确率排名 | Precision | TP / (TP + FP)，基于金标准 |
| 召回率排名 | Recall | TP / (TP + FN)，基于金标准 |
| 综合排名 | F1 | 2 × P × R / (P + R) |
| 独立验证排名 | CRISPR验证率 | 有spacer证据的边占比 |
| 稳定性排名 | Bootstrap一致性 | 1000次重采样后边的保留率 |

### 4.4 Bootstrap置信区间

- 对整合网络进行1,000次有放回重采样
- 记录每条边在重采样中的出现频率
- 计算边的稳定性指数（edge stability index）：出现频率 ≥ 0.95的边定义为"高稳定边"

## 五、生态信号挖掘与统计建模

### 5.1 生态位特异性分析

识别不同生态环境中特异性的噬菌体-宿主互作关系：

**方法**：
- 将噬菌体-宿主互作边按来源推断的生态位分组（肠道/海洋/多环境）
- 使用Fisher精确检验检验每条边是否在某一生境中显著富集
- 多重检验校正：Benjamini-Hochberg FDR校正，显著性阈值 q < 0.05

**输出**：
- 肠道特异性互作列表（enriched in gut）
- 海洋特异性互作列表（enriched in ocean）
- 泛生境互作列表（ubiquitous interactions）

### 5.2 生活方式-生态关联分析

检验温和噬菌体（temperate）与烈性噬菌体（virulent）在不同宿主属中的分布是否存在显著差异：

**方法**：
- 对每个宿主属（噬菌体数 ≥ 100），构建2×2列联表（生活方式 × 生态位）
- 卡方检验（Chi-squared test），计算Cramér's V效应量
- 多重检验校正：Bonferroni校正

**关键假设**：
- 温和噬菌体在稳定生境（如肠道）中相对富集（因为溶原策略在稳定环境中更有利）
- 烈性噬菌体在动态生境（如海洋）中相对富集

### 5.3 网络模体分析与零模型检验

识别噬菌体-宿主互作网络中的重复性拓扑模式：

**网络指标**：
- **嵌套度（Nestedness）**：使用NODF（Nestedness metric based on Overlap and Decreasing Fill）量化
- **模块化（Modularity）**：使用Louvain算法检测网络社区结构
- **特化程度（Specialization）**：使用H2'指标量化噬菌体-宿主互作的特化程度

**零模型检验**：
- 生成1,000个随机二分网络，保持度分布不变（使用Patefield算法或曲棍球算法）
- 计算观察值与零模型分布的标准化效应大小（SES）
- 检验观察到的嵌套度、模块化和特化程度是否显著偏离随机预期

### 5.4 互作网络β多样性分析

比较不同生态位之间噬菌体-宿主互作网络的差异：

**方法**：
- 计算生态位间的Jaccard距离（基于互作边的存在/缺失）
- 计算Bray-Curtis距离（基于互作边的丰度/置信度加权）
- PERMANOVA检验：检验生态位间互作网络组成是否存在显著差异
- PCoA（主坐标分析）可视化生态位间的互作网络差异

### 5.5 稀释曲线分析（Rarefaction Analysis）

评估对主要宿主属的互作空间采样是否饱和：

**方法**：
- 对每个主要宿主属，逐步增加采样的噬菌体数量
- 记录新发现的互作关系数量
- 拟合Michaelis-Menten模型估计互作空间的渐近大小
- 评估当前采样是否接近饱和（观察到的互作数 / 估计总互作数）

## 六、CRISPR Spacer深度分析模块

### 6.1 Spacer提取流水线

从宿主基因组中系统性提取CRISPR阵列：

**数据源**：
- NCBI RefSeq中覆盖PhageScope Top 50宿主属的完整基因组
- 预计涉及数百个参考基因组

**提取工具**：
- 使用CRISPRCasFinder（v4）或MinCED进行CRISPR阵列检测
- 提取每个阵列中的所有spacer序列（通常30-50bp）
- 去冗余：使用CD-HIT-EST对spacer序列进行聚类（99% identity阈值）

**质控标准**：
- 仅保留符合CRISPR阵列特征（重复序列保守性、间隔序列长度一致性）的检测结果
- 记录每个基因组的CRISPR阵列数量、spacer总数、重复序列类型

### 6.2 噬菌体-原间隔序列匹配

将提取的spacer序列与567K+噬菌体序列进行系统性比对：

**比对策略**：
- BLASTn：短序列优化参数（word_size=7, reward=1, penalty=-3, gapopen=5, gapextend=2）
- 匹配阈值：identity ≥ 95%，alignment length ≥ 28bp，E-value < 1e-5
- 考虑PAM（Protospacer Adjacent Motif）验证：在匹配位点侧翼检查已知的PAM序列

**计算规模**：
- 预计spacer数量：10万-50万条
- 目标序列数量：567K+
- 需要高性能计算集群支持（预计计算量：10^11-10^12 次碱基对比较）

### 6.3 历史互作重建

Spacer作为宿主免疫记忆的分子化石，可用于重建历史互作：

**分析思路**：
- 同一宿主属内不同菌株的spacer多样性反映了长期噬菌体捕食压力
- 使用spacer共享网络（spacer-sharing network）推断噬菌体群落的历史动态
- 与PhageScope当前的预测互作网络进行比较，检验"历史互作"与"当前预测"的一致性

### 6.4 与预测互作的交叉验证

核心问题：PhageScope预测的互作关系中有多少比例有独立的CRISPR证据支持？

**分层分析**：
- 按数据源计算CRISPR验证率
- 按宿主属计算CRISPR验证率
- 按MECS评分区间计算CRISPR验证率（预期：高MECS → 高CRISPR验证率）
- 使用逻辑回归模型检验MECS评分是否能预测CRISPR验证结果

## 七、实验验证数据集的构建与整合

### 7.1 金标准数据集编纂

目标：构建≥5,000对实验验证的噬菌体-宿主对，作为网络可信度评估的基准

**数据来源**：

| 来源 | 预计配对数 | 验证等级 |
|---|---|---|
| PhagesDB | ~3,000+ | 高（培养分离验证） |
| ATCC | ~500+ | 高（商业标准化验证） |
| DSMZ | ~300+ | 高（商业标准化验证） |
| 文献检索 | ~1,000-2,000 | 中-高（培养验证但有方法差异） |

### 7.2 数据库整合

**整合流程**：
1. 从各数据库API/下载接口获取原始配对数据
2. 统一噬菌体ID映射（通过NCBI accession或基因组序列比对）
3. 统一宿主分类学映射（基于NCBI Taxonomy的统一命名）
4. 去冗余：合并来自多个来源的重复配对，记录证据多重性

### 7.3 分类学统一化

- 使用NCBI Taxonomy作为参考分类系统
- 将PhageScope中的宿主注释和实验验证数据集中的宿主注释映射到同一属级分类
- 对无法映射或分类学有争议的配对进行标注，在敏感性分析中分别处理

### 7.4 性能基准测试

使用金标准数据集对以下对象进行性能评估：

| 评估对象 | 描述 | 预期结果 |
|---|---|---|
| 单源网络 | 每个数据源单独构建的互作网络 | 识别最可靠的数据源 |
| 整合网络 | 所有来源合并的网络 | 评估简单合并的效果 |
| 过滤网络 | 仅保留MECS高置信度边的网络 | 评估过滤策略的增益 |
| CRISPR验证网络 | 仅保留有CRISPR证据支持的边 | 评估独立验证的严格程度 |

## 十一、方法学权衡与透明度声明

### 11.1 敏感性与特异性的权衡

- **高敏感性策略**：保留所有来源的预测结果，最大化互作网络的覆盖范围，但会引入大量假阳性
- **高特异性策略**：仅保留多来源一致支持且CRISPR验证的互作，最小化假阳性，但会遗漏大量真实互作
- **本研究的平衡点**：通过MECS评分提供连续的置信度梯度，允许下游用户根据研究需求自行选择阈值

### 11.2 来源包容性与网络可靠性的权衡

- **全源合并**：保留所有8个来源，最大化数据量（567K记录），但低可靠性来源可能污染网络
- **选择性合并**：仅纳入F1排名前3-4的来源，提高网络质量，但减少覆盖范围
- **本研究的策略**：先进行全源分析，再通过性能基准测试识别可靠来源，最终提供两种网络（全源 + 高可靠源）

### 11.3 生态位推断的局限性

- **核心局限**：本研究从数据源推断生态位（如GPD → 肠道），而非使用噬菌体本身的直接环境元数据
- **影响**：部分噬菌体可能来自非典型生境，导致生态位分类错误
- **透明度措施**：在结果中明确标注推断生态位 vs 确认生态位；对IMG_VR等覆盖多环境的来源，单独分析其内部的生态位分布

### 11.4 CRISPR匹配的局限性

- **系统偏差**：CRISPR系统仅存在于约50%的细菌和90%的古菌中，不含CRISPR系统的宿主无法通过此方法验证
- **历史偏差**：Spacer记录的是历史互作，当前噬菌体可能已进化逃逸
- **技术偏差**：Spacer长度短（30-50bp），可能导致假阳性匹配
- **透明度措施**：报告CRISPR覆盖率统计量；对无CRISPR数据的宿主属，仅依赖实验金标准验证

### 11.5 多重检验校正的权衡

- **BH FDR校正**：控制FDR，但在大规模检验（数万条边）下可能过度保守
- **Storey q-value**：替代方案，估计真正的零假设比例，可能提供更宽松的阈值
- **本研究策略**：主分析使用BH FDR（q < 0.05），补充分析使用Storey q-value，比较两种方法的一致发现

## 十二、总结

本研究方案以PhageScope的873,718条噬菌体记录为基础，构建了一个从数据预处理到生态信号挖掘的完整分析框架。其核心贡献在于：

1. **不是简单地合并多源数据**，而是系统评估每个来源的可靠性，并提供基于多证据的置信度评分
2. **不是盲目信任计算预测**，而是通过CRISPR spacer和实验验证两个独立证据层进行交叉验证
3. **不是停留在网络描述**，而是通过零模型检验和生态位分层分析，区分真实生态信号与方法学假象

最终成果将包括一个来源可靠性排名的互作网络、一套生态位特异性的互作图谱、一个CRISPR验证的高置信度子集，以及一个开源的多源网络可信度评估管线。这些成果将为噬菌体疗法、微生物组工程和噬菌体生态学提供可靠的数据基础。
